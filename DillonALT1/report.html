3.1 (c) Evidence of Computational Thinking:

Decomposition

I broke down the test into levels with increasing difficulties. At the end of each level, the programme determines whether the user has passed the level or not. One big problem I ran into was that if a user failed, the loop would have them retrying forever and ever until they passed. Through decomposition, I altered the program so that it allows the user to exit the game if they decide they have given up.

Pattern recognition

During my research, I noticed a pattern in the similiarities of each memory game I looked to for inspiration. Almost every one included a level which tested the reflexes of an individual and/or tested the ability of the user to retain information.
Given this pattern, it indicated that these approaches were the most popular to accurately determine the users reactions.

Abstraction

By allowing the users to decide when they have reached their ability and giving them the power to end the game when they wish, I have ultimately separated them into 3 different classes:

  • Level 1 standard reactions
  • Level 2 standard reactions
  • Level 3 standard reactions
This means that any outliers in the data would have been abstracted and put into a class with similiar ability, which would allow me to further investigate the data according to the ability of the individuals. Therefore, I could evaluate the results more in depth for a highly accurate result.
