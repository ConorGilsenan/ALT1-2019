1. A Rationale for the Approach to the Brief

1.1 Research

Before beginning our project we decided to investigate different websites to get an idea of what we can do for our own website. We then decided to create a game and specifically a reaction game.


1.2 Response to Brief

We decided to make a reaction speed game. We went with the most basic design as we wanted to make it as easy to use as possible. This game can be played by anyone that wants to test their reactions

3. Design and Development of the Artefact

3.1(a) The iterative design process

Version 0.1 was just a blank screen with each heading.
Version 0.5 included the new fonts and colours
Version 1.0 had our first design of the reaction game
Version 1.5 connected our database to the website and the game had a timer at the bottom
Version 1.7 had a start screen for the game so it wouldnt start straight away
Version 1.9 added a progression bar at the top of the game
Version 2.0 the score that you got is sent away to a database
Version 2.5 the highest scores are displayed on the side
Version 3.0 allowed the user to input their name
Version 3.5 sorted the highest score correctly and displayed the users name beside it
Version 3.8 corrected the size of text and position of everything
Version 4.0 (Final product) shapes are now sized randomly between 70 pixels and 100 pixels

3.1(b) Algorithm

3.1(c) Evidence of Computational Thinking

Through Decomposition, we broke down the creation of our website into three basic parts:
1. It has a working game
2. It sends the user's score to a database
3. It reads the scores from the database and displays them

3.2 Development of the Artefact

We decided on having very little on the page as it makes the page much more understandible and easier to navigate. This meets many of the headings of UX Design e.g. Usability First and Less Is More
The website also allows the user to input their own name. This fits under the heading: The User Is In Control

4. Evaluation of the artefact

4.1 Artefact features

It has a fully functional game that has no errors
It successfully sends the user's score to a database
It allows the user to add their name for the leaderboard
There are no problems with expressing the top scores from the database
The game successfully makes the shapes a random size between 70px and 100px
We were unable to add other shapes as it would make the code for pressing them very long and complicated

4.2 Future Development

We would like to try and figure out a code that would allow us to use more shapes that are not 4 sided
We would also like to have the game display where you are located in the score breakdown
